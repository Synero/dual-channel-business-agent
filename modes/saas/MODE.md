# Mode: SaaS

**For:** Software companies that sell subscriptions, tiers, and digital services.

## Examples
- B2B SaaS platforms (CRM, project management, invoicing)
- Apps with freemium/paid tiers
- API services, data providers
- No-code/low-code tools
- Hosting, domains, cloud services
- Digital products (courses, templates, plugins)

## Customer Intent
The customer wants to:
1. Understand what the product does and if it fits their need
2. Know pricing tiers and what's included in each
3. Book a demo or start a free trial
4. Get technical support or onboarding help
5. Upgrade, downgrade, or cancel their subscription

## Conversation Flow
```
Customer: "Hola, que hace su plataforma?"
→ Brief intro, ask about their use case to tailor the pitch

Customer: "Necesito algo para gestionar mi equipo de 10 personas"
→ Match to the right tier, highlight relevant features

Customer: "Quiero ver la posibilidad de una demo"
→ Confirm trial length, explain trial limitations, send signup link

Customer: "Tengo un problema con la integracion"
→ Ask for details, give troubleshooting steps, escalate if needed

Customer: "Quiero subir de plan"
→ Confirm new tier, explain billing change, process upgrade
```

## Key Data
- **Product Tiers:** Plans, prices, features per tier
- **Trials & Demos:** Active trials, demo bookings, conversion status
- **Support Tickets:** Issues reported, status, resolution
- **Customer Memory:** Company size, use case, current tier, support history

## Payment Patterns
- Monthly/annual subscription (recurring)
- Usage-based billing (per seat, per API call, per GB)
- One-time setup fee
- Enterprise custom pricing (contact sales)

## What Makes This Mode Unique
- **Product is intangible** — customer can't touch it, needs trust
- **Trial matters** — free trial is the main conversion tool
- **Support is ongoing** — relationship doesn't end at sale
- **Technical depth** — customers ask integration/API questions
- **Churn risk** — need to proactively check satisfaction
